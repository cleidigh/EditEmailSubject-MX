Edit Email Subject: Local Mode Deprecation

Since version 1.0, the default operation mode used by the Edit Email Subject add-on for changing the subject was to modify the local message database. This mode did not actually change the message.

After IMAP servers became more popular, a new mode was introduced, which changed the subject in the message itself. This change was synchronized with the IMAP server and was therefore visible on all other IMAP clients connected to the same email account.

During the last year it became obvious, that the local mode causes confusions:

  • the original subject is restored when message is moved/copied (applies to messages in local folders, POP folders and IMAP folders)
  • the original subject is restored when folder is repaired (applies to messages in local folders, POP folders and IMAP folders)
  • most of the time users are confused that the subject is not updated on all devices (applies to messages in IMAP folders)

We therefore decided to drop support for the local mode and no longer modify the message database, but always update the message. Technically, the message is re-created with a new UUID and the updated subject. The original message is moved to a local folder named "EES-Temp", in case something went wrong.
